# Insight — 跨书沉淀池

对话里长出来的、值得回头看的东西。
冷的时候、纠结的时候、又陷在同一个模式里的时候，回到这里。

**跨书**：所有书共用这一份。同一段经历可能被多本书照见——
那正是这个结构想要的，反链会自然显出"原来一直是同一件事"。

## 5 个维度

每个维度一个文件夹，每条 insight 一个 `.md`。
**所有条目的骨架在 [`INDEX.md`](INDEX.md)**——反向路径（用户带新经历来）从那里开始扫。

| 文件夹 | 装什么 | 什么时候打开 |
|---|---|---|
| [`概念/`](概念/) | 核心概念的工作定义 + 用户理解的演化轨迹 | 想回温某个概念，或看自己理解怎么变了 |
| [`你的故事/`](你的故事/) | 用户真实经历被书照见的部分 | 又陷在同一个模式里时；想重温某段经历的理解 |
| [`闪回/`](闪回/) | 用户经历 × 书的观点 → 融汇出的洞察、分析、结论（事实层） | 想再走一遍那次"哦，原来"的连接 |
| [`共振/`](共振/) | 闪回的更深一层——融汇击中了用户当下的感受 | 想找回那种"原来一直在等这句话"的瞬间 |
| [`悬题/`](悬题/) | 当下不解、带着读下去的问题 | 检查哪些问题还在跟着你走 |

## 引用约定

**书 → 章**：`[[书名:章节]]`（例：`[[道德经:ch02]]`）
**条目 → 条目**：`[[条目名]]`（例：`[[抓]]` 引用 `概念/抓.md`）
**跨维度引用**：`[[维度/条目]]`（例：`[[闪回/那个声音同一份恐惧]]`，避免重名歧义时使用）

不需要建专门的反链页——文件里搜 `[[书名:` 或 `[[条目名]]` 可找到所有相关条目。

## 文件命名约定

| 维度 | 文件名 |
|---|---|
| `概念/` | `<概念名>.md`，如 `道.md` |
| `你的故事/` | `<事件slug>.md`，如 `加密币的那一年.md` |
| `闪回/` | `<洞察主题slug>.md`，如 `那个声音同一份恐惧.md` |
| `共振/` | `<日期>-<slug>.md`，如 `2026-05-28-那个声音.md` |
| `悬题/` | `<问题slug>.md`，如 `无为变成新策略.md` |

每个条目文件第一行：`> 来自 [[书名:章节]] · 创建于 YYYY-MM-DD`。
后续被另一章 / 另一本书再次照见，在引用行追加新的 `[[书名:章节]]`。

## 新维度

如果对话长出来一种东西，不属于上面 5 类，但也值得记下的，告诉用户，询问是否新建维度，由用户来判断。
