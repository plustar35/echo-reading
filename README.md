# echo-reading

跟 Claude Code 一起**深读一本书**——逐段读、说出理解、跟 LLM 聊下去，让书照见自己的人生。

读到的字，是手指；字指向的认知，才是月亮。echo-reading 的全部目的是：**在 Claude Code 的陪读下，把一本书真正读透、内化成自己的东西**。调动你的人生经历、听书在你身上的回声，是为了让书里的认知真的落地——不是反过来把"挖掘人生"当目的。

## 这不是什么

- 不是读书笔记自动生成器（让 AI 替你"总结一本书"）
- 不是 Anki / 知识管理工具
- 不是聊天 chatbot

## 这是什么

一个交给 Claude Code 维护的对话式深读笔记本。你每次读完一段，说出你的理解、卡点、联想。LLM 围绕你的肉身经验和章节内容讨论，**帮你读透**。讨论中自然冒出的闪光点——某个概念忽然有了私人定义、某段经历被书"咔哒"对上、某个问题没法当下解开——LLM 静默写到 `insight/` 对应维度下。

主任务是"读透"；insight 是副产品。**颠倒这个顺序就跑歪了**。

## 工作原理

一本书从"一个文件"走到"你自己的认知"，中间是这样一条路：

```
   📖 电子书                        ┌──────────────── books/ 原文层（事实，永不改写）────────────────┐
  epub/mobi/azw3                    │                                                                │
        │                           │   chNN/raw.md         chNN/01.md  02.md …                      │
        │   ① book-ingest           │   ┌──────────┐        ┌─────────────────────┐                  │
        └──────────────────────────▶│   │  纯原文  │───────▶│ 阅读单元 · 9 段骨架 │                  │
            提取 + 按章切            │   │（底本）  │  ②     │ 原文灌满            │                  │
            留档源文件              │   └──────────┘ chapter│ 字词注/直译/注家…   │                  │
                                    │                -split │ ⑧ 你的理解（留白）  │                  │
                                    └───────────────────────┴──────────│──────────┴──────────────────┘
                                                                        │ ③ 逐段深读
                                                                        ▼
                                              ┌─────────────────────────────────────────┐
                                              │            主线 · 帮你读透               │
                                              │                                          │
                                              │    你：说出理解 / 卡点 / 联想            │
                                              │            ⇅  围绕你的肉身经验讨论       │
                                              │  Claude：照见、解惑、把字句缝回经验      │
                                              └────────────────────┬─────────────────────┘
                                                                   │ ④ 讨论中自然冒出闪光点
                                                                   │   （副线：浮现才收，绝不为收而问）
                                                                   ▼
   ┌──────────────────────── insight/ 衍生层（跨所有书共享）────────────────────────┐
   │   概念/        你的故事/        闪回/         共振/            悬题/             │
   │  工作定义     经历被照见    经历×观点的洞察  击中感受的瞬间   带着走的问题      │
   └────────────────────────────────────────────────────────────────────────────────┘
```

一句话：**原文只进不改（左上→中），理解在对话里长出来（中），闪光点静默沉淀到 insight（下）**。读透是目的，沉淀是回声。

## 怎么开始

前提：装好 [Claude Code](https://claude.com/claude-code)。

```bash
git clone https://github.com/<your-user>/echo-reading.git my-reading
cd my-reading
claude
```

然后**把一本电子书丢给 Claude**（epub / mobi / azw3 都行）：

> 导入这本书 〔附上电子书文件〕

Claude 会调用 `book-ingest` skill：先看清这本 epub 的结构，判断怎么切，把每一章的**纯原文**提取出来，在 `books/<书名>/` 下按「一章一目录」生成 `chNN/raw.md`，同时写出 `progress.md`、把源文件留档进书目录。入库只到原文就位为止。切完告诉你共几章、可以读第几章。

> 没有电子书也行：直接说「读《xxx》第 1 章」，Claude 会手工建这一章的 `raw.md`。

接着开始读：

> 读《xxx》第 1 章

如果这一章还只有 `raw.md`（没备读过），Claude 会先调用 `chapter-split` skill 把章节原文切成若干**阅读单元**——短章铺成一个 `01.md`，长章切成多个单元再加一份卷层面的 `00-导读.md`，每个单元都是固定的 9 段结构（原文灌满、注疏/译文先占位、第 8·9 段留白）。备好后开始读第一个单元，补齐字词注 / 白话直译 / 注家对照 / 三个值得想的问题，等你贴出"自己的理解"。讨论开始后，闪光点会被静默收集到 `insight/` 下。

继续：

> 下一章 / 深究一下 / 我今天又遇到一件事，跟之前那段聊过的有关吗

## 目录结构

```
books/<书名>/            ← 原文层。永不修改。
  ├── progress.md        阅读进度（章 → 阅读单元 两层）
  ├── <源文件>.epub      入库时留档的电子书源文件
  └── chNN/              一章一目录
        ├── raw.md         这章纯原文（入库产物，只读底本）
        ├── 00-导读.md     长章才有——卷/章层面入口
        └── NN.md          一个阅读单元一文件，9 段固定结构

insight/                 ← 衍生层。对话里长出来的沉淀。跨所有书共享。
  ├── README.md          维度说明 + 引用约定
  ├── INDEX.md           主干——所有条目骨架，每条一行
  ├── 概念/              核心概念的工作定义
  ├── 你的故事/          用户真实经历被书照见
  ├── 闪回/              用户经历 × 书的观点 融汇出的洞察
  ├── 共振/              击中感受/情绪的洞察
  └── 悬题/              当下不解、带着读下去的问题
```

详细规则全部写在 [`CLAUDE.md`](CLAUDE.md) 和 [`insight/README.md`](insight/README.md) 里——Claude Code 启动时会自动加载。

## 设计原则

- **两层分离**：`books/` 是事实层（不写衍生分析）；`insight/` 是衍生层（不写原文）
- **跨书共享**：所有书共用一份 `insight/`，同一段经历可能被多本书照见
- **浮现而非匹配**：值得收的东西靠判断识别，不靠匹配某个条件
- **理解为主，记录为副**：颠倒就走偏

## 为什么叫 echo-reading

回声式阅读——你把理解说出去，书和 LLM 把回声送回来。你拿自己的人生去叩书，书才会用你听得懂的方式，把它的认知真正回给你：不是停在字面，而是接住了、变成你自己的那一层。
