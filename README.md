# echo-reading

跟 Claude Code 一起**深读一本书**——逐段读、说出理解、跟 LLM 扯下去，让书照见自己的人生。

读到的字，是手指。书想给你看的，是月亮。
echo-reading 的全部目的是：**让书把你的人生回声给你听**。

## 这不是什么

- 不是读书笔记自动生成器（让 AI 替你"总结一本书"）
- 不是 Anki / 知识管理工具
- 不是聊天 chatbot

## 这是什么

一个交给 Claude Code 维护的对话式深读笔记本。
你每次读完一段，说出你的理解、卡点、联想。LLM 围绕你的肉身经验和章节内容讨论，**帮你读透**。
讨论中自然冒出的闪光点——某个概念忽然有了私人定义、某段经历被书"咔哒"对上、某个问题没法当下解开——LLM 静默写到 `insight/` 对应维度下。

主任务是"读透"；insight 是副产品。**颠倒这个顺序就跑歪了**。

## 怎么开始

前提：装好 [Claude Code](https://claude.com/claude-code)。

```bash
git clone https://github.com/<your-user>/echo-reading.git my-reading
cd my-reading
claude
```

然后跟 Claude 说：

> 读《道德经》第 1 章

Claude 会读 `CLAUDE.md` 里的工作流，在 `books/道德经/` 下建好这一章的笔记骨架（原文 / 字词注 / 白话直译 / 注家对照 / 三个值得想的问题 ...），等你贴出"自己的理解"。
讨论开始后，闪光点会被静默收集到 `insight/` 下。

继续：

> 下一章 / 深究一下 / 我今天又遇到一件事，跟之前那段聊过的有关吗

## 目录结构

```
books/<书名>/            ← 原文层。永不修改。
  ├── progress.md        阅读进度
  └── chNN.md            一章一文件，9 段固定结构

insight/                 ← 衍生层。对话里长出来的沉淀。跨所有书共享。
  ├── README.md          维度说明 + 引用约定
  ├── INDEX.md           主干——所有条目骨架，每条一行
  ├── 概念/              核心概念的工作定义
  ├── 你的故事/          用户真实经历被书照见
  ├── 闪回/              用户经历 × 书的观点 融汇出的洞察
  ├── 共振/              击中感受/情绪的洞察
  └── 悬题/              当下不解、带着读下去的问题
```

详细规则全部写在 [`CLAUDE.md`](CLAUDE.md) 和 [`insight/README.md`](insight/README.md) 里——Claude Code 启动时会自动加载。

## 设计原则

- **两层分离**：`books/` 是事实层（不写衍生分析）；`insight/` 是衍生层（不写原文）
- **跨书共享**：所有书共用一份 `insight/`，同一段经历可能被多本书照见
- **浮现而非匹配**：值得收的东西靠判断识别，不靠匹配某个条件
- **理解为主，记录为副**：颠倒就走偏

## 为什么叫 echo-reading

回声式阅读——你把理解说出去，书和 LLM 把你**说的内容的回声**送回来。
听见的不是书的内容，是你自己在书的形状里说话的样子。
