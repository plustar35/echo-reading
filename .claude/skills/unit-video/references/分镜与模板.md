# 分镜与模板 —— 把口播切成 beats

`分镜.js` 是每个单元唯一要产出的数据文件，通用渲染器 `scripts/renderer.html` 读它出片。起手不用从零写：`draft-storyboard.js` 已生成骨架——narr 按口播稿逐字切好、T0 按看板就绪时刻预分、每拍标了真实起止秒。本文讲怎么把骨架**填完**：选模板、拆经文、给点亮锚、写左看板。

> 本文写的是**当前渲染器的模板系统**——五套模板的「皮」（看板材质、几何、配色、字体）由素材套及其配置定（见 `资产层与新书.md`）；「beats 切分、模板分配、字幕/看板对齐」这套机制对任何素材套不变。

## 一、渲染器既定事实（只读；分镜只填字，不碰布局）

- **视觉底层全书复用**：前段是片头视频，之后背景循环无限播。看板在片头里 `scrollReadyMs`（见素材套 `配置.js`）才就绪，此前没法往上写字。
- **16:9 课件四区**：

  ```
  ┌──────────────── 标题区(章名,卷轴右缘竖排) ────────────────┐
  │  讲师位          ┌──── 看板 = 卷轴 ──────────────┐        │
  │  (作者,左下,     │  左:辅助看板    右:经文(逐句)  │        │
  │   固定,朝卷轴)   └──────────────────────────────┘        │
  │ ─────────────────── 字幕区(口播,按句) ─────────────────── │
  └──────────────────────────────────────────────────────────┘
  ```

- **排版钉死在 renderer 的 CSS 里**：经文统一字号、整块在卷轴里上下居中、竖排从顶起右→左铺、长句自动分列；注释 / 标题文字左对齐。配色：卷轴浅色绢帛、经文墨黑、点亮朱砂、字幕浅字。
- **字幕跟模板解耦**：字幕逐字取自每拍的 `narr`，renderer 按标点自动切成单行短句、在该拍时间内依次走——字幕永远是子句粒度的完整短句；卷轴上显什么、何时换版式，由 beat 决定。

## 二、五套模板（不为某一章新建第六套）

| 模板 | 版式 | 用在什么拍 |
|---|---|---|
| **T0 开场白** | 纯字幕浮在片头画面上，不碰卷轴 | 只用在卷轴铺好前的开场引入 |
| **T1 原文+注** | 右：经文逐句呈现 / 点亮 ｜ 左：结构化小看板 `aux` | 逐句精讲（主力） |
| **T2 标题+引** | 居中大标题 `title` + 1~2 句 `lead[]`，无经文 | 引子、过渡 |
| **T3 中央大字** | 卷轴正中短大字 `glyph`（8 字以内、每行 ≤4 字，书法体、特大、居中） | 题眼 / 章核心字；最适合卷轴刚揭开那一刻 |
| **T4 一句话** | 一句大字 `big`（铺满卷轴、靠左，可 `<br>`/`<em>`） | 收束金句、反问、点睛 |

T3 与 T4 的区别：T4 是一句话，T3 是题字短词——8 字以内、每行 ≤4 字、最多两行（align 硬校验）；超 4 字必须用 `<br>` 在语义断点自己断，自动折行断点会错。

模板原则：

- **左看板能独立读懂**：`head` 说清这镜在讲什么 + 完整短句 `points`，像 PPT 一页，不是丢碎词。
- **长排比挑重点注、不堆全、不编号**：如「七善」用 T1，左注只挑要害。
- **经文逐句呈现**：讲到第几句，卷轴显第几句。

## 三、怎么填：模板分配与拍子增减

第一道闸是时间（骨架已按它预分），第二道才是内容：

- **T0**（骨架已标）：卷轴铺好前的开场。T0 的结束时刻不得越过 `scrollReadyMs + 800ms`——align 硬校验，越线即报错中止，绝不渲出「卷轴铺好却空着」的尬白。
- **卷轴揭开的首拍**：口播正落在题眼字词上 → **T3** 把它「泼」出来，与旁白同步，是开场→卷轴最顺的缝合法（骨架把骑线句在子句标点处切开，正是为这一手）；不落在题眼字上就用 T2。非 T0 拍起点早于 `scrollReadyMs` 没关系——看板内容显示时刻 = max(拍起点, scrollReadyMs)，渲染器自动钳位，之前只走该拍字幕。
- 之后看"那一刻口播在讲什么"：引子 / 过渡 → **T2**；逐句精讲 → **T1**；收束金句 → **T4**。

**拍子可并可拆，拆点落在子句标点（，：；。——）上**：字幕按标点走完整子句，切在词中间会让字幕冒出半截短语（align 对此打软提醒，人审时确认）。长排比（一连串同构句念 20 多秒）可在「；」处切成两三拍、`hi` 逐段点亮，比单拍 `sweep` 干站更有节奏。

**只能动切分，不能动文字**：所有 narr 顺序拼接必须逐字等于口播稿（align 字符流硬校验，唯二容差：标点形态、注音括号「字（pinyin）」）。要改内容，回口播稿重跑配音。

**密度参考**：开场 1–2 条 T0（+1 条 T3 承接）；引子 1–2 条 T2；一句原文常对应 2~5 条 T1，每条点亮该句的一个短语、配一块看板；收束 1–2 条 T4。每拍落在第几秒永远以时间轴为准，密度只管「切几刀、各放什么」。

## 四、schema

一个赋值给 `STORYBOARD` 的对象（末尾两行 `window.STORYBOARD=` / `module.exports=` 照抄）。素材不写路径——`assets` 一填，管线自动去 `SKILL/assets/<素材id>/` 取（不填则缺省用与 `book` 同名的套）。`audio` 相对分镜所在目录，直接写文件名。

```js
const STORYBOARD = {
  book:"<书名>", chapter:"chNN/NN",                     // 内容元数据
  assets:"<书名>/<素材id>",                             // 素材套定位键（SKILL/assets/ 下相对路径）
  scrollTitle:"《书名》·第 N 章",                       // 卷轴右缘竖排小字
  audio:"口播.tts.m4a",                                 // gen-tts 生成，分镜旁边

  jing:[                                                // 卷轴经文：逐句；{k,t}=可点亮短语，{p}=标点
    { id:"s1", parts:[ {k:"a",t:"〈经文短语〉"}, {p:"。"} ] },
    { id:"s2", parts:[ {k:"b1",t:"〈短语〉"},{k:"b2",t:"〈短语〉"},{p:"。"} ] },
  ],

  beats:[                                               // 一根时间轴：顺着口播稿，narr 逐字喂 TTS+字幕
    { seg:"片头", tpl:"T0", narr:"〈开场……我只给你一个字：〉" },
    { seg:"片头", tpl:"T3", glyph:"水", narr:"〈水。〉" },          // 卷轴揭开首拍
    { seg:"引子", tpl:"T2", title:"〈标题〉", lead:["〈引句〉"], narr:"〈引子旁白〉" },
    { seg:"〈段名〉", tpl:"T1", sent:"s1", hi:["a"],
      aux:{ head:"〈这镜在讲什么〉", gloss:[{z:"〈字〉",m:"〈释义〉"}], points:["〈完整短句〉"] },
      narr:"〈讲这句经文的旁白〉" },
    { seg:"收束", tpl:"T4", big:"〈一句大字〉<br>〈可 <em>朱砂强调</em>〉", narr:"〈收束旁白〉" },
  ],
  durs:[],                                              // 留空数组，align-durs.js 回填
};
if (typeof window!=="undefined") window.STORYBOARD=STORYBOARD;
if (typeof module!=="undefined") module.exports=STORYBOARD;
```

**字段速查**：`seg` 段名（同名连续拍归一组，给进度点用）｜ `tpl` 模板 ｜ `sent` 经文句 id（仅 T1）｜ `hi` 点亮锚（仅 T1）｜ `aux{head, gloss:[{z,p,m}], points:[]}` 左看板 ｜ `title`/`lead[]`（T2）｜ `glyph`（T3）｜ `big`（T4）｜ `narr` 旁白（每拍必有，T0 也是）。各模板的必填字段与引用完整性由 align 校验。

**点亮锚 `hi`**：看该拍 `narr` 正讲到经文哪个字，在 `jing` 里找出那字所在短语的 key 填进去。

- `hi:["b3"]` —— 点亮 key 为 `b3` 的短语；`hi:["a1","a2"]` —— 多个一起点亮（想让半句单独点亮，就在 `jing` 里把这句拆成两个 `{k}` 段）。
- `hi:"sweep"` —— 整句逐字扫亮（成组排比整拍呈现时用）。
- 省略 `hi` —— 不点亮，整句平铺。

`gloss` 每条：`z`＝字、`m`＝释义（必填）；`p`＝拼音只有多音字 / 生僻字才写。`gloss`/`points`/`lead` 没有就整个省掉，别留空数组。

可选覆盖：`scrollReadyMs`（毫秒）——一般不写，单一来源是素材套的 `配置.js`；只有这一个单元要特调时才在分镜里覆盖。
