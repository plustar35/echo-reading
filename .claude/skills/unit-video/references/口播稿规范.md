# 口播稿规范 —— 一条领读怎么讲

口播稿只管一件事：**这条领读怎么讲，才顺、才自然、才能讲透**。把所有旁白从上往下连起来读，就是完整的一条领读。模板、点亮、秒数全是下游分镜对着配音时间轴定的（见 `分镜与模板.md`），写稿时心里别装那些。

**素材**：从备读 `books/<书>/chNN/NN.md` 第 1–7 段提炼；第 8/9 段（用户留白）不进视频。

## 一、voice：你就是作者本人在带读

- **作者第一人称**。道德经＝老子，论语＝孔子，理想国＝苏格拉底……以书的作者 / 主讲人身份开口，像老师当面带着「你」读，不是旁白解说。
- **只对「你」说**。通篇第二人称，不说「我们」「读者」「大家」。
- **大白话、密、不学术腔**。一句顶一句，不灌水、不掉书袋。注家的观点化进白话里讲，不报家数（不说「王弼认为」「河上公注」）、不铺「三家对照」的目录——那是备读第 4 段干的事，视频要「听得进去」。

## 二、结构：一段连续口播

0. **开场**：报书名章号 + 一句钩子。习惯写法「今天，我们读《X》第 N 章」，不强制。
1. **引子**：接上一章 / 前文，把「上次到哪、这次看什么」带出来，免得一上来就砸原文。
2. **逐句精讲**（主体）：顺着原文一句一句讲透，详略不平均、重点深讲，以讲懂原文为主线。
3. **收束**：收一句金句 / 反问 / 点睛，停在内容的「劲」上。

## 三、每句怎么讲（四拍）

引出原文 → 大白话讲透（顺口点一下关键字什么意思）→ 落到当代经验 → 点出这句的「劲」。不是每句都凑满四拍，但「讲透 + 落地 + 点劲」是骨头。

## 四、文言读音校对（送 TTS 前必做）

TTS 不认文言多音字。送配音前逐句过一遍读音，列一张读音校对表写在口播稿开头（首个 `---` 之前）。常见坑：处 chǔ / 恶 wù / 几 jī / 尤 yóu / 乐 yuè / 王 wàng（动词）……通假、生僻字拿不准就标。

落法：在旁白里把字写成「处（chǔ）」——`gen-tts.js` 剥掉括号注音只念字，字幕里保留帮观众。

## 五、parse 契约（gen-tts 怎么读这份稿）

- 首个 `---` 之前放标题、voice 说明、读音校对表，不进配音。
- 之后**每组连续的 `>` 引用行 = 一条旁白段**；空行 / `##` 标题行 / `---` 断段（只是排版，不影响内容）。
- **每段旁白以句末标点（。！？）收尾**——各段直接相接送 TTS，缺了句子会跨段粘连（gen-tts 强校验）。
- `>` 行逐字喂 TTS + 字幕，只放要讲出口的话，不夹给下游看的备忘。
